# /generate PRP [feature-name]

## Objectif
Créer un PRP court, exécutable, <1200 tokens.

## Contenu attendu
- 🎯 Objectif utilisateur en 1–2 phrases.
- 📦 Contexte technique: fichiers et modules impactés.
- 🧩 Description fonctionnelle succincte (3–6 puces).
- ⚙️ Implémentation technique ciblée:
  - Server Actions, routes, composants, hooks Prisma
  - contrats d’entrée/sortie typés (DTO)
- 🔗 Ressources techniques minimales:
  - clés du `knowledge_registry.json` strictement nécessaires
- 🧪 Tests/Validation: 2–4 assertions clés
- 🔗 Intégration Front: composants à brancher, props ou context à propager
- ✅ Done When: critères mesurables

## Règles
- Aucun texte redondant. Pas de digressions.
- Ne référencer que les ressources nécessaires à cette feature.
